# Лабораторна робота №5

## Створення системи автентифікації

### Тема

Створення системи автентифікації. Реалізація форм входу/реєстрації та захищених маршрутів (Protected Routes) за допомогою Context API.

---

# Мета роботи

Навчитися використовувати **Context API** для глобального керування станом у React-додатку, реалізувати систему авторизації (вхід, реєстрація, вихід), а також створити **захищені маршрути (Protected Routes)** для обмеження доступу до приватних сторінок.

---

# 1. Реалізація AuthContext

Файл: `src/context/AuthContext.jsx`

```javascript
import { createContext, useState } from "react";

// створення контексту
export const AuthContext = createContext();

// створення провайдера
export const AuthProvider = ({ children }) => {

  const [isAuthenticated, setIsAuthenticated] = useState(false);
  const [user, setUser] = useState(null);

  // функція входу
  const login = (userData) => {
    setIsAuthenticated(true);
    setUser(userData);
  };

  // функція виходу
  const logout = () => {
    setIsAuthenticated(false);
    setUser(null);
  };

  return (
    <AuthContext.Provider
      value={{
        isAuthenticated,
        user,
        login,
        logout
      }}
    >
      {children}
    </AuthContext.Provider>
  );
};
```

Цей контекст використовується для збереження інформації про авторизацію користувача та надає доступ до функцій входу і виходу у будь-якому компоненті додатку.

# 2. Реалізація ProtectedRoute

Файл: `src/components/hoc/ProtectedRoute.jsx`

```javascript
import { Navigate, Outlet } from "react-router-dom";
import { useContext } from "react";
import { AuthContext } from "../../context/AuthContext";

const ProtectedRoute = ({ redirectPath = "/login", children }) => {

  const { isAuthenticated } = useContext(AuthContext);

  if (!isAuthenticated) {
    return <Navigate to={redirectPath} replace />;
  }

  return children ? children : <Outlet />;
};

export default ProtectedRoute;
```

Компонент `ProtectedRoute` перевіряє стан авторизації користувача.
Якщо користувач не авторизований, виконується перенаправлення на сторінку входу.
Якщо авторизація є, відображається відповідна сторінка.

# Контрольні запитання

## 1. Яку архітектурну проблему вирішує використання Context API?

Context API вирішує проблему **prop drilling**.
Prop drilling виникає тоді, коли потрібно передавати дані через багато рівнів компонентів за допомогою пропсів. Це ускладнює структуру додатку та робить код менш читабельним.

Context API дозволяє створити **глобальний стан**, який може використовуватися будь-яким компонентом без необхідності передавати пропси через усі рівні компонентів.

---

## 2. Чому для глобального управління станом у складних додатках іноді використовують Redux або Zustand?

У великих додатках Context API може викликати зайві перерендери компонентів та ускладнювати управління складною логікою стану.

Бібліотеки **Redux** або **Zustand** надають:

* більш структуроване управління станом
* централізоване зберігання даних
* інструменти для відлагодження (DevTools)
* middleware для роботи з асинхронними операціями
* кращу масштабованість у великих проектах

Тому для складних систем управління станом часто використовують спеціалізовані бібліотеки.

---

## 3. Яка роль патерна Higher-Order Component (HOC) при реалізації захищених маршрутів?

Higher-Order Component (HOC) — це функція або компонент, який приймає інший компонент і повертає новий компонент з додатковою логікою.

У випадку **ProtectedRoute** HOC використовується для:

* перевірки авторизації користувача
* обмеження доступу до певних сторінок
* перенаправлення неавторизованих користувачів на сторінку входу

Таким чином HOC дозволяє повторно використовувати логіку захисту маршрутів у різних частинах додатку.

---

## 4. Чому при перенаправленні використовується `replace: true`?

Використання `replace: true` у компоненті `Navigate` замінює поточний запис у **стеку історії браузера**.

Це означає, що сторінка, з якої було виконано перенаправлення, не зберігається в історії.
Якщо користувач натисне кнопку **"Назад"**, він не повернеться на сторінку, доступ до якої був обмежений.

Це допомагає уникнути ситуацій, коли користувач може повернутися на захищену сторінку через кнопку "Back" у браузері.
