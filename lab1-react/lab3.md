## 2. Просте умовне відображення (оператор `&&`)

Було додано кнопку для відображення або приховування довідки про програму.

Використано стан:

```jsx
const [showHelp, setShowHelp] = useState(false);
```

Кнопка для перемикання стану:

```jsx
<button onClick={() => setShowHelp(!showHelp)}>
  {showHelp ? "Приховати інструкцію" : "Показати інструкцію"}
</button>
```

Умовний рендеринг довідки:

```jsx
{showHelp && <p>Довідка: Дозволяє керувати списками студентів.</p>}
```

## 3. Тернарний оператор (Toggle)

Реалізовано перемикач режиму перегляду списку студентів.

Створено стан:

```jsx
const [filterActive, setFilterActive] = useState(false);
```

Кнопка для перемикання режиму:

```jsx
<button onClick={() => setFilterActive(!filterActive)}>
  {filterActive ? "Показати всіх" : "Показати тільки успішних"}
</button>
```

Відображення списку:

```jsx
{(filterActive ? filteredStudents : allStudents).map(student => (
  <div key={student.id}>
    {student.name}

    <span style={{ color: student.score >= 60 ? "green" : "red" }}>
      {student.score >= 60 ? "Зараховано" : "Незараховано"}
    </span>
  </div>
))}
```

## 4. Реалізація інтерфейсу з табами

Було створено систему вкладок:

* Всі студенти
* Статистика
* Про автора

Стан активної вкладки:

```jsx
const [activeTab, setActiveTab] = useState("list");
```

Навігація між вкладками:

```jsx
<button onClick={() => setActiveTab("list")}>Всі студенти</button>
<button onClick={() => setActiveTab("stats")}>Статистика</button>
<button onClick={() => setActiveTab("about")}>Про автора</button>
```

Логіка відображення:

```jsx
<div className="content">
  {activeTab === "list" && <StudentList />}
  {activeTab === "stats" && <StatisticsData />}
  {activeTab === "about" && <AboutAuthor />}
</div>
```

# Контрольні запитання

### 1. Чому `if/else` не можна використовувати прямо в JSX?

JSX підтримує тільки вирази, а не інструкції. `if/else` є інструкцією, тому їх потрібно використовувати поза JSX або замінювати тернарним оператором.

---

### 2. Яка проблема з використанням `0 &&` у JSX?

Якщо в лівій частині `&&` знаходиться `0`, React відобразить `0` на екрані, тому що `0` є значенням, яке може бути відрендерене.

---

### 3. Як реалізувати складний умовний рендеринг (5+ варіантів)?

Краще використовувати:

* `switch`
* об'єкт-мапу компонентів
* окремі функції або компоненти

---

### 4. Коли краще використовувати тернарний оператор?

Коли існує **два взаємовиключні варіанти відображення**.

---

### 5. Як React розуміє, що потрібно перерендерити компонент?

При зміні **state або props** React запускає процес повторного рендерингу компонента.
