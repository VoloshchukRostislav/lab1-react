# Практична робота №8

## Абстракція бізнес-логіки. Кастомні хуки

---

## Мета

Навчитися відокремлювати бізнес-логіку від UI-компонентів, створювати кастомні хуки та повторно використовувати логіку.

---

## Використані технології

* React
* Vite
* JavaScript

---

## Хід роботи

### 1. Проблема дублювання логіки

У різних компонентах може повторюватися однаковий useEffect (наприклад resize або online/offline).
Це ускладнює підтримку коду.

---

### 2. Хук useWindowSize

Було створено хук:

```jsx
const { width, height } = useWindowSize();
```

Він:

* слухає resize
* повертає розмір вікна

---

### 3. Хук useOnlineStatus

```jsx
const isOnline = useOnlineStatus();
```

Він:

* використовує navigator.onLine
* реагує на online/offline

---

### 4. Власний хук useLocalStorage

```jsx
const [value, setValue] = useLocalStorage("key", initialValue);
```

Він:

* зберігає дані в localStorage
* автоматично оновлює їх через useEffect

---

### 5. Використання

Було реалізовано:

* адаптивність (mobile view)
* банер offline
* збереження імені користувача

---

## Відповіді на контрольні питання

### 1. Правило кастомних хуків

Хук має починатися з `use`.
Це потрібно для коректної роботи React Hooks та ESLint.

---

### 2. Чи спільний стан?

Ні.
Кожен виклик хука — це окремий стан.

---

### 3. Чому [] у useEffect

Щоб виконати один раз.
Без нього ефект виконувався б при кожному рендері.

---

### 4. Чому важливий cleanup

Без removeEventListener:

* накопичуються обробники
* виникає memory leak
* падає продуктивність

---

### 5. HOC vs Hooks

Обидва вирішують повторне використання логіки.

Hooks краще, тому що:

* простіші
* менше вкладеності
* читабельніші

---

## Висновок

Було створено кастомні хуки для повторного використання логіки, що значно покращує архітектуру та масштабованість додатку.
